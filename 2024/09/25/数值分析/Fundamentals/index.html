2. Binary Numbers

2.1 Decimal to Binary

  • Integer Part: Convert decimal integers to binary by dividing by 2 successively and recording the remainders.
  • Fractional Part: Multiply by 2 successively and record the integer parts, moving away from the decimal point to the right.

2.2 Binary to Decimal

  • Integer Part: Simply add up powers of 2.
  • Fractional Part: If the fractional part is finite (a terminating base 2 expansion), proceed the same way. When the fractional part is not a finite base 2 expansion, use the shift property of multiplication by 2.

3. Floating Point Representation of Real Numbers

3.1 Floating Point Format

The models in modern use are based on the IEEE 754 Floating Point Standard, which consists of a set of binary representation of real numbers.

A floating point number consists three parts: sign, mantissa, exponent. There are three commonly used levels of precision: single, double, and extended.
![[image-${date}.png]]

The form of a normalized IEEE floating point number is

±1.bbb…b×2p\pm 1.bbb\dots b\times 2^p ±1.bbb…b×2p

where each of the N b’s is 0 or 1, and p is an M-bit binary number representing the
exponent. Normalization means the leading (leftmost) bit must be 1. When a binary number is stored as a normalized floating point number, it is “leftjustified,” meaning that the leftmost 1 is shifted just to the left of the radix point.

Single and long double precision are handled in the same way, but with different choices for M and N.

Def (Machine Epsilon): The number machine epsilon, denoted ϵmach\epsilon_{mach}ϵmach​, is the distance between 1 and the smallest floating point number greater than 1. For the IEEE double precision floating point
standard,

ϵmach=2−52\epsilon_{mach}=2^{-52} ϵmach​=2−52

Due to the machine epsilon, we can not represent a digit in any precision, we must truncate the number in some way, and in so doing we necessarily make a small error. There are two ways of truncation:

  1. Chopping: simply throw away the bits that fall off the end—that is, those beyond the 52nd bit to the right of the decimal point.
  2. Rounding: add 1 to bit 52 (round up) if bit 53 is 1, and to do nothing (round down) to bit 52 if bit 53 is 0, with one exception: If the bits following bit 52 are 10000… , exactly halfway between up and down, we round up or round down according to which choice makes the final bit 52 equal to 0. (IEEE Rounding to Nearest Rule)

Def: Denote the IEEE double precision floating point number associated to xxx, using the Rounding to Nearest Rule, by fl(x)fl(x)fl(x).

The floating point representation of a real number is found by the following procedure:

  1. Justify: Shift radix point to the right of the leftmost 1, and compensate with the exponent.
  2. Round: Apply a rounding rule, such as the IEEE Rounding to Nearest Rule, to reduce the mantissa to 52 bits.
